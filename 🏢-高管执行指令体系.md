## ⚠️ 第六步：问题升级路径

### 【如果遇到问题】怎么处理

```
问题分类 → 升级路径 → 处理时间

小问题（P2）：
  发生 → 项目经理 → 开发负责人 → 本日解决
  
中问题（P1）：
  发生 → 项目经理 → 开发负责人 + 高管通知 → 次日解决
  
严重问题（P0）：
  发生 → 项目经理 → 立即报告高管 → 会议确认方案 → 1小时内解决

示例：

问题1：iPhone上表单显示异常
  严重度：P1
  报告链：QA → 项目经理 → 开发负责人
  处理：当天解决
  
问题2：Server酱推送失败
  严重度：P0
  报告链：运维 → 项目经理 → 立即报告高管
  处理：1小时内排查恢复
  
问题3：某个字段的验证逻辑需要调整
  严重度：P2
  报告链：QA → 项目经理 → 开发负责人
  处理：本日解决，可与其他改动一起部署
```
