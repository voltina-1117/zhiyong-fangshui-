## 🔴 一级问题：业务流程漏洞（致命）

### 1. 【严重】重复提交防控机制不完善
**现状：** 
- 仅在客户端检查 `10分钟内不超过1次`
- 靠手机号判断，无法识别使用者身份
- 同一个人可用多个手机号提交多条意向单

**业务风险：**
- ❌ 竞争对手可恶意投放虚假号码拖垮系统
- ❌ 客户可批量自己提交虚假意向单来占用接单人员时间
- ❌ 抖音作弊团队可自动化脚本刷单

**优化方案：**
```
【方案A】基于设备指纹（推荐）
- 获取设备UA、IP、屏幕分辨率、字体库等生成设备ID
- 同设备24小时内仅可提交3条意向单
- 超出部分进入审核队列（标记"疑似重复"）

【方案B】基于IP地址段
- 同IP /24网段24小时内最多10条
- 同IP每2小时最多3条

【方案C】结合人工审核
- 提交时自动标记风险等级（低/中/高）
- 高风险单据进入人工审核队列
- 审核员1分钟内确认真假
```

**实施优先级：** ⭐⭐⭐⭐⭐ 第一期必须做

### 2. 【严重】缺少抖音渠道特殊处理

- 无任何针对抖音生态的优化

### 5. 【严重】双管理员协作机制不清
**现状：**
- 文档说"双人跟进"，但代码无法体现
- 权限完全相同，无分工
- 无法追踪谁在处理哪个单据

**业务风险：**
- ❌ 一个客户可能被两个人都联系，造成重复骚扰
- ❌ 同一单据两人都可能在操作，导致数据冲突
- ❌ 无法评估每个人的工作量和效率
- ❌ 无法明确责任分工

**优化方案：**
```
改进管理模式：
1. 引入"认领机制"
   - 单据状态新增"待认领"
   - 管理员A/B 看到新单时要"认领"
   - 只有认领人才能操作这个单据
   - 30秒内没人认领，自动推送提醒

2. 引入"接力机制"
   - 某管理员忙碌时，可以"转交"给另一人
   - 转交必须确认，不能强制转
   - 系统记录转交原因和时间

3. 权限细分
   - A权限：可以修改自己认领的单据
   - B权限：可以查看全部，但只修改自己的
   - 管理权限：可以转交他人单据、删除、编辑管理员账号

4. 工作报表
   - 每日统计：谁认领了多少单、完成了多少单
   - 每周对比：A vs B 的完成率、转化率
   - 识别瓶颈
```

## 🟠 二级问题：数据验证与安全（高危）

### 7. 【高危】电话号码验证不完整
**现状：**
```javascript
if (!/^1[3-9]\d{9}$/.test(data.phone))
```
**问题：**
- ✅ 长度正确（但格式死板）
- ❌ 没有验证号码是否真实存在
- ❌ 没有检测虚拟运营商
- ❌ 没有识别国际漫游号
- ❌ 没有检查黑名单

**优化方案：**
```
const validatePhone = async (phone) => {
  // 1. 格式校验（现有）
  if (!/^1[3-9]\d{9}$/.test(phone)) return false;
  
  // 2. 虚拟号识别
  const virtualPrefixes = ['170', '171', '162', '167']; // 虚拟运营商
  if (virtualPrefixes.some(p => phone.startsWith(p))) {
    markRisk('virtual_operator');
  }
  
  // 3. 黑名单检查（本地缓存）
  if (blacklist.has(phone)) return false;
  
  // 4. 三要素验证（调用API，可选）
  // 需要付费第三方API，但能排除90%的虚假号码
  try {
    const valid = await checkPhoneAPI(phone);
    if (!valid) markRisk('invalid_phone');
  } catch (e) {
    // API失败时不阻断，但标记为高风险
    markRisk('phone_not_verified');
  }
  
  return true;
};
```

**实施优先级：** ⭐⭐⭐⭐ 第一期建议做

### 8. 【高危】姓名字段无验证
**现状：**
```javascript
if (!data.name) { setErr('name', '请填写联系人姓名'); ok = false; }
```

**问题：**
- ✅ 非空检查
- ❌ 接受 "测试"、"111"、"xxxxxxxxxxx" 这样的无效名字
- ❌ 没有中文字符验证（国际用户？）
- ❌ 没有长度限制（可输入几千字符造成DB问题）

**优化方案：**
```javascript
function validateName(name) {
  const trimmed = name.trim();
  
  // 长度检查
  if (trimmed.length < 2 || trimmed.length > 20) {
    return { valid: false, msg: '姓名长度2-20字符' };
  }
  
  // 黑名单关键词（筛选测试单）
  const blacklist = ['测试', '试试', '123', 'test', 'admin', '删除'];
  if (blacklist.some(word => trimmed.includes(word))) {
    return { valid: false, msg: '姓名包含非法关键词' };
  }
  
  // 字符类型检查：允许中文、英文、数字、空格、·、-
  if (!/^[\u4e00-\u9fff\u3400-\u4dbfa-zA-Z0-9\s·\-]{2,20}$/.test(trimmed)) {
    return { valid: false, msg: '姓名格式不正确' };
  }
  
  return { valid: true };
}
```

**实施优先级：** ⭐⭐⭐ 第一期做

### 9. 【高危】缺少CSRF/XSRF防护
**现状：**
- 表单直接POST到API
- 没有CSRF令牌
- 任何网站的JS都可能伪造表单提交

**业务风险：**
- ❌ 竞争对手网站可嵌入表单，代客户提交虚假意向单
- ❌ 黑产可批量自动化提交垃圾数据

**优化方案：**
```javascript
// 前端生成CSRF令牌
async function getCSRFToken() {
  const existing = sessionStorage.getItem('csrf_token');
  if (existing) return existing;
  
  const response = await fetch('/api/csrf-token');
  const { token } = await response.json();
  sessionStorage.setItem('csrf_token', token);
  return token;
}

// 提交时必须携带
form.addEventListener('submit', async (e) => {
  e.preventDefault();
  const csrfToken = await getCSRFToken();
  
  const res = await fetch(cfg.CREATE_LEAD_URL, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'X-CSRF-Token': csrfToken,
    },
    body: JSON.stringify(data),
  });
});
```

**实施优先级：** ⭐⭐⭐⭐ 第一期必须做

```javascript
        Object.assign({}, data, { source: getSource() })

      leadNo = lead.lead_no;
    } else {
      var res = await fetch(cfg.CREATE_LEAD_URL, {
        method: 'POST',
        headers: {
          'Content-Type': 'application/json',
          'Authorization': 'Bearer ' + cfg.SUPABASE_ANON_KEY,
        },
        body: JSON.stringify(Object.assign({}, data, { source: getSource() })),
        signal: submitController.signal,
      });
      var json = await res.json();
      if (!res.ok) throw new Error(json.error || '提交失败');
      leadNo = json.lead_no;
    }

    progressText.textContent = '✓ 提交成功！';
    clearTimeout(timeout);
    setTimeout(() => {
      location.href = 'success.html?lead_no=' + encodeURIComponent(leadNo);
    }, 500);
  } catch (err) {
    clearTimeout(timeout);
    if (err.name === 'AbortError') {
      progressText.textContent = '已取消';
      tipText.textContent = '提交已取消';
      setTimeout(() => {
        submitProgress.classList.add('hidden');
        submitBtn.hidden = false;
      }, 1000);
    } else {
      progressText.textContent = '❌ 提交失败';
      tipText.innerHTML = err.message + '<br><button class="btn-retry">点击重试</button>';
      document.querySelector('.btn-retry').onclick = () => form.dispatchEvent(new Event('submit'));
    }
  }
});

document.getElementById('cancelSubmit').addEventListener('click', () => {
  if (submitController) submitController.abort();
});
```

**实施优先级：** ⭐⭐⭐ 第一期做

```javascript
    const hash = require('crypto').createHash('md5')
      .update(navigator.userAgent + testName)
```
