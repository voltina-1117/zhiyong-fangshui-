# 详细设计说明书

> 对应需求： [00-软件需求规格说明书.md](./00-软件需求规格说明书.md)

## 1 系统结构

```
[抖音/微信浏览器]
       │
       ├─ index.html ──→ create-lead ──→ Supabase
       │                      │
       │                      └─→ Server酱 ×2
       │
       └─ admin.html ──→ admin-api ──→ Supabase
```

## 2 客户留资页

页头：品牌 + 服务专线（tel） + 距离收费说明  
表单：姓名、手机、位置、紧急程度、描述、协议  
提交：校验 → 写库 → 跳转 success.html

## 3 管理端

登录 → 列表（优先级+时间） → 详情（拨号、查勘报价、状态按钮）  
前台轮询 30s；新单主依赖 Server酱。

## 4 状态机

意向待联系 → 已联系 → 已预约查勘 → 已查勘 → 方案已报价 → 维修中 → 待平台付款 → 已完成

已完成须 payment_status = paid。

## 5 优先级映射

| urgency 值 | priority |
|------------|----------|
| 紧急（需尽快处理） | 1 |
| 较急（希望这几天看） | 2 |
| 一般（可预约时间） | 3 |

## 6 文件清单

| 文件 | 职责 |
|------|------|
| index.html / success.html | 客户侧 |
| admin.html | 管理侧 |
| js/form.js | 留资提交 |
| js/admin.js | 管理逻辑 |
| js/demo-store.js | 本地预览数据 |
| supabase/functions/* | 服务端 |
