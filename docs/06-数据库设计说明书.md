# 数据库设计说明书

> 基线：[00-前期需求确认.md](./00-前期需求确认.md)

## 6.1 leads 表

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | uuid | 自动 | 主键 |
| lead_no | text | 是 | YX20260710-001 |
| name | text | 是 | 姓名 |
| phone | text | 是 | 电话 |
| location | text | 是 | 位置 |
| urgency | text | 是 | 紧急/较急/一般（完整文案存库） |
| priority | int | 是 | 1/2/3 |
| description | text | 是 | 大致描述 |
| source | text | 是 | douyin 等 |
| status | text | 是 | 意向待联系… |
| survey_note | text | 否 | 查勘备注 |
| solution | text | 否 | 方案 |
| quote_amount | numeric | 否 | 报价 |
| final_amount | numeric | 否 | 应收 |
| payment_status | text | 否 | unpaid/paid |
| paid_at | timestamptz | 否 | 收款时间 |
| last_operator | text | 否 | 最后操作的管理员 display_name |
| created_at | timestamptz | 自动 | |
| updated_at | timestamptz | 自动 | |

## 6.2 admin_users 表（2 条记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid | 主键 |
| username | text | 登录名，唯一 |
| password_hash | text | bcrypt 等 |
| display_name | text | 留痕显示名 |
| created_at | timestamptz | |

第一期：**2 个账号，权限相同**，不做角色表。

## 6.3 urgency → priority

| urgency | priority |
|---------|----------|
| 紧急（需尽快处理） | 1 |
| 较急（希望这几天看） | 2 |
| 一般（可预约时间） | 3 |

## 6.4 状态枚举

意向待联系、已联系、已预约查勘、已查勘、方案已报价、维修中、待平台付款、已完成、已取消

**已完成** 须 `payment_status = paid`。

## 6.5 操作留痕

管理端每次改 status / 报价 / 确认收款，更新 `last_operator` 为当前登录人 `display_name`；可选增 `lead_logs` 表（第二期）。
