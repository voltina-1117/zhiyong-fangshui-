# 测试计划

## 8.1 测试范围

- 表单输入校验
- 照片上传功能
- 数据提交与存储
- 页面兼容性
- 二维码扫码可用性
- 紧急联系拨号

## 8.2 测试用例

| 编号 | 测试项 | 操作步骤 | 预期结果 |
|------|--------|----------|----------|
| TC-01 | 必填字段校验 | 不填姓名，点击提交 | 提示「请输入姓名」 |
| TC-02 | 手机号格式校验 | 输入「123」，点击提交 | 提示「请输入正确的手机号」 |
| TC-03 | 正常提交 | 填写所有必填项，点击提交 | 跳转成功页，数据库新增记录 |
| TC-04 | 照片上传 | 选择一张照片 | 显示缩略图，云存储有文件 |
| TC-05 | 照片数量限制 | 连续选 4 张 | 提示「最多上传3张」 |
| TC-06 | 照片大小限制 | 选择 >5MB 图片 | 提示文件过大 |
| TC-07 | 紧急联系拨号 | 点击底部电话 | 弹出拨号界面 |
| TC-08 | 微信内打开 | 微信扫码打开 | 页面与功能正常 |
| TC-09 | 浏览器打开 | Safari/Chrome 打开 | 页面布局正常 |
| TC-10 | 防重复提交 | 提交过程中多次点击按钮 | 仅提交一次 |
| TC-11 | 成功页返回 | 点击「返回首页」 | 回到 index.html |
| TC-12 | 数据库字段 | 提交后查看 orders 记录 | 字段完整，status=待受理 |
| TC-13 | 高优先级 | 勾「大面积渗水」提交 | priority=1，列表排前 |
| TC-14 | 中优先级 | 只勾「墙体发霉」 | priority=2 |
| TC-15 | 低优先级 | 只勾「其他」 | priority=3 |
| TC-16 | 混合优先级 | 同时勾「墙体发霉」+「滴水成线」 | priority=1（高覆盖中） |
