# 包工头/平台端管理方案

> 你是**唯一运营**，管理「意向单」全流程直至平台收款。  
> 基线：[00-前期需求确认.md](./00-前期需求确认.md)

## 1. 你的日常流程

```
Server酱：抖音来新意向单
    ↓
admin.html → 🔴 高优先级先看
    ↓
打电话 → 改「已联系」→ 约查勘
    ↓
上门看完 → 「已查勘」→ 填方案+报价
    ↓
开工 → 「维修中」→ 完工 → 「待平台付款」
    ↓
客户平台付完款（第一期：你确认到账）→ 「已完成」
```

## 2. 入口

| 用途 | 链接 |
|------|------|
| 抖音挂给客户 | `index.html?from=douyin` |
| 你自己管理 | `admin.html`（收藏，勿公开） |

## 4. 管理端：2 人同权限

- 2 个独立账号登录（如包工头 + 接单员）
- 权限相同：看全部单、改状态、报价、确认收款
- 每次操作记录 `last_operator`
- Server酱 推新单（一个或两个通道均可）

典型分工：接单员首电+约查勘；包工头查勘+报价+施工+确认款。

## 4. 平台付款（两阶段）

| 阶段 | 做法 |
|------|------|
| 第一期 | 修完后改「待平台付款」，客户按平台规则付款，你在系统里「确认到账」 |
| 第二期 | 发 `pay.html` 链接，微信/支付宝在线付，自动改已完成 |

## 5. 业务红线

- 维修款**必须经平台**才能结案（系统状态约束）
- 报价在查勘后录入，客户留资页**不展示价格**
