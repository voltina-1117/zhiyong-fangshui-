## 1 引言

### 1.1 编写目的

本文档描述「志勇防水意向单管理系统」雏形应实现的功能范围、业务流程及验收标准，供股东演示、开发与下一阶段上线使用。

### 1.2 项目背景

业务通过抖音等渠道获取防水维修意向客户。客户在线留下联系方式并选择**大致问题类型**；两名内部人员（志勇师傅 / 副手）电话跟进、预约查勘、报价施工；款项经平台结算。

### 1.3 术语说明

| 术语 | 含义 |
|------|------|
| 意向单 | 客户在线提交的预约记录 |
| 问题类型 | 客户从预设列表中选择的大致漏水/渗水类别 |
| 查勘 | 工作人员上门查看现场 |
| 平台收款 | 维修费用经平台确认或在线支付 |

### 1.4 参考文档

- 《部署与手机配置说明》SETUP.md  
- 《数据库设计说明》docs/06-数据库设计说明书.md

## 3 功能需求

### 3.1 客户留资页（index.html）

#### 3.1.1 页面布局

1. 品牌区：志勇防水  
2. 服务专线：13022222222，可点击拨号  
3. 费用说明：查勘交通费按实际距离收取  
4. 预约表单  
5. 页脚：服务时间 8:00—20:00  

#### 3.1.2 表单字段

| 字段 | 必填 | 说明 |
|------|------|------|
| 手机号码 | 是 | 11 位大陆手机号 |
| 所在位置 | 是 | 小区/街道/门牌 |
| **问题类型** | **是** | 下拉单选，见 3.1.3 |
| 问题描述 | 是 | 自由文本 |
| 联系人姓名 | 否 | 折叠区「更多信息（可选）」 |
| 紧急程度 | 否 | 三选一，不选则按问题类型定优先级 |
| 同意条款 | 是 | 勾选后提交 |

进度条显示 **4 项必填** 完成度（手机 / 位置 / 问题类型 / 描述）。

**禁止**为姓名、紧急程度填充默认值。

#### 3.1.3 问题类型选项

- 卫生间/厨房漏水  
- 屋面/顶层漏水  
- 外墙/窗户渗雨  
- 地下室/潮湿发霉  
- 楼下投诉/邻里纠纷  
- 不确定，需先看现场  

#### 3.1.4 优先级规则（雏形）

1. 若客户选择了紧急程度 → 按紧急程度映射 1/2/3  
2. 未选紧急程度 → 按问题类型默认：  
   - 「楼下投诉/邻里纠纷」→ 1  
   - 「不确定，需先看现场」→ 3  
   - 其余 → 2  

#### 3.1.5 提交与成功页

- 单号格式：`YX` + 年月日 + `-` + 三位序号  
- 成功页展示单号；一般单提示 24 小时内联系；若选了「紧急」则提示 2 小时内优先联系  
- 同一手机号 1 分钟内仅允许提交一次（便于客户补充说明后重新提交）  
- URL 参数 `from` 记录来源（如 `douyin`）

### 3.2 管理端（admin.html）

#### 3.2.1 登录

- 账号 `admin1` / `admin2`，密码 `changeme123`（雏形演示用）  
- 登录后显示 **志勇师傅** / **副手**  
- 副标题：**志勇防水 · 内部工作台**

#### 3.2.2 列表

- 排序：优先级升序，同优先级按时间降序  
- 状态筛选 Tab  
- 列表项：单号、优先级、姓名（未留名则显示「未留名」）、脱敏电话、**问题类型**、位置、描述摘要、状态、经办人  

#### 3.2.3 详情

- 展示 `problem_type`、紧急程度（未指定则显示「未指定」）  
- 拨号、查勘记录、报价、状态流转至「已完成」  
- 每 30 秒自动刷新（前台时）

#### 3.2.4 新单通知（上线后）

Server酱 双通道推送；雏形阶段不启用。

### 3.3 意向单状态流转

```
意向待联系 → 已联系 → 已预约查勘 → 已查勘 → 方案已报价
    → 维修中 → 待平台付款 → 已完成
```

任一步可进入「已取消」。「已完成」须从「待平台付款」进入并标记已收款。

## 4 本地预览（DEMO_MODE）

1. 用浏览器打开项目根目录 `index.html`  
2. 填写表单并提交 → 跳转 `success.html`  
3. 打开 `admin.html`，使用 `admin1` / `changeme123` 登录  
4. 顶部黄条「预览模式」表示数据保存在本机 localStorage  

关闭预览模式：在 `js/config.js` 配置 Supabase 地址并将 `DEMO_MODE` 设为 `false`，见 SETUP.md。
