# 手机端与管理端通知方案

> 问题：两个包工头都要在**手机**上看单，最好「有单就提示」。  
> 基线：[00-前期需求确认.md](./00-前期需求确认.md)

---

## 一、先讲结论（说真话）

| 想法 | 能不能实现 | 说明 |
|------|------------|------|
| 管理页**适配手机** | ✅ 能 | admin.html 移动端布局，大按钮、大字号 |
| 两人**各自手机**登录看单 | ✅ 能 | 2 个账号，各登各的 |
| H5 **一直挂后台**自动刷新 | ❌ **不可靠** | iOS/安卓会杀后台网页，无法保证常驻 |
| 有单**立刻推送到手机** | ✅ 能 | 靠 **微信推送**，不是靠网页挂后台 |
| 点通知**跳进管理页看详情** | ✅ 能 | 推送里带链接 |

**正确思路：** 不要指望浏览器后台挂着；用 **Server酱 → 微信消息** 当「来电铃」，需要看单时再点开管理页。

---

## 二、推荐方案（第一期 · 0 元 · 最稳）

```
客户提交意向单
      ↓
Supabase create-lead 成功
      ↓
同时推 2 条 Server酱（包工头 A、包工头 B 各一个 SendKey）
      ↓
两人微信收到消息（锁屏、后台、没开浏览器都能收）
      ↓
点消息里的链接 → 打开 admin.html（已登录或需输一次密码）
      ↓
看详情 → 拨号 → 改状态
```

### 两人怎么配通知

| 方式 | 做法 | 推荐 |
|------|------|------|
| **各配各的** | 张师傅一个 SendKey，李师傅一个 SendKey，新单推两次 | ⭐ **推荐** |
| 共用一个 | 一个 SendKey 绑一个微信，另一人靠口头转达 | 不推荐 |
| 企业微信群 | Webhook 推群里（需企业微信） | 有公司再用 |

### 推送内容示例

```
【新意向单】🔴紧急
YX20260710-003
张三 138****8000
位置：XX区XX小区
描述：卫生间滴水3天
👉 点击处理：https://xxx.vercel.app/admin.html?id=xxx
```

## 三、管理页在手机怎么用（第一期）

| 步骤 | 操作 |
|------|------|
| 1 | 微信收到 Server酱 消息 |
| 2 | 点链接打开管理页（抖音/微信内置浏览器均可） |
| 3 | 首次输入账号密码，勾选「记住 30 天」 |
| 4 | **添加到主屏幕**（见下），下次像 App 一样点图标进 |

### 添加到主屏幕（像 App 一样）

**iPhone（Safari 或微信里用浏览器打开后）：**

- 分享 →「添加到主屏幕」→ 起名「防水接单」

**安卓：**

- Chrome 菜单 →「添加到主屏幕」 / 「安装应用」

> 这是快捷入口，**不是**后台常驻；通知仍靠微信 Server酱。

## 四、管理页打开时的小增强（第一期可做）

当 admin.html **正在前台打开**时：

| 能力 | 作用 |
|------|------|
| 每 30 秒自动刷新列表 | 页面亮着时补漏（Server酱 为主） |
| 新单提示音 | 检测到新 lead_no 播放短音 |
| 震动（安卓部分浏览器） | 同上 |

**仅限页面在前台**，后台仍会被系统挂起。

## 五、第二期可选：PWA + 系统通知（进阶）

| 项 | 说明 |
|----|------|
| PWA | manifest.json + Service Worker，「安装到桌面」 |
| Web Push | 浏览器系统级通知（Android 较好；iOS 16.4+ 有限制） |
| 成本 | 仍 0 元，但开发量比 Server酱 大 |
| 建议 | **第一期不做**，Server酱 够用了再考虑 |

## 六、不推荐的方案

| 方案 | 为什么不推荐 |
|------|--------------|
| 让网页 7×24 挂后台 | 手机省电策略会杀进程，不可靠 |
| 只做轮询、不做推送 | 关屏就漏单 |
| 短信通知 | 要钱，第一期不必 |
| 为了推送去做小程序 | 你已定不做小程序；微信推送用 Server酱 更简单 |
| 两人共用一个密码 | 无法留痕谁操作的 |

## 七、第一期需求补充（写入开发）

| 编号 | 功能 |
|------|------|
| M-01 | admin.html **移动端优先**布局（375px 基准） |
| M-02 | 列表卡片大点击区、详情底部固定操作栏 |
| M-03 | Server酱 **双 SendKey**，新单同时推 2 人 |
| M-04 | 推送带 **admin 深链**（?id= 或 ?lead_no=） |
| M-05 | 登录 **记住 30 天**（localStorage token） |
| M-06 | 前台时 **30 秒轮询** + 可选提示音 |
| M-07 | 提供「添加到主屏幕」说明（用户手册） |

**不做：** 真·后台常驻、PWA 系统推送（第二期候选）。

## 八、两人日常 SOP

```
新单 → 两人微信同时响
    ↓
谁先看到谁处理（或约定接单员首响）
    ↓
点链接进管理页 → 拨号 → 改状态（系统记 last_operator）
    ↓
另一人打开列表也能看到已更新状态
```

## 九、配置清单（上线前）

| 配置 | 谁准备 |
|------|--------|
| 包工头 A 的 Server酱 SendKey | 人 A 在 sct.ftqq.com 申请 |
| 包工头 B 的 Server酱 SendKey | 人 B 申请 |
| 两个 SendKey 写入 Supabase 环境变量 | 开发/部署时 |
| 管理页 HTTPS 链接 | Vercel 部署后 |
| 两人各存主屏幕快捷方式 | 各自手机操作 |

> Server酱 免费档对新单量小的生意通常够用；超限再考虑 PushPlus 等同类服务。
